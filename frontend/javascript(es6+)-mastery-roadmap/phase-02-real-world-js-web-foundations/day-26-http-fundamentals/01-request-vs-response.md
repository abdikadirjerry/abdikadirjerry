# Day 26 — Request vs Response

## Request Info

Method: GET  
URL: https://jsonplaceholder.typicode.com/posts  
Headers:

- Accept: application/json
- User-Agent: Browser

## Response Info

Status Code: 200  
Content-Type: application/json  
Response Data Type: JSON (Array of objects)

## My Notes

A Request is sent by the browser to the server asking for data or action.

A Response is what the server sends back, including:

- Status (success or error)
- Data (JSON, HTML, etc.)

So:
Client → Request → Server  
Server → Response → Client
